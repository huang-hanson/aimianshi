# [并发] 线程池任务执行流程

## 一、题目分析

### 考察点
- 线程池的核心参数及作用
- 任务执行流程（7 个步骤）
- 线程池的饱和策略（拒绝策略）
- 任务队列的作用
- 线程池的状态转换

### 难度等级
⭐⭐⭐⭐⭐ (5/5 星) - 高频核心难题

### 适用岗位
- 中级/高级 Java 开发工程师
- 所有 Java 并发相关岗位必考题

---

## 二、核心答案

**一句话回答：**
> 任务提交后，线程池按以下流程执行：**1. 核心线程数未满 → 创建核心线程执行**；**2. 核心线程已满 → 放入工作队列**；**3. 队列已满 → 创建非核心线程执行**；**4. 线程数达最大值 → 执行拒绝策略**。

### 3.2 线程池核心参数

```
┌─────────────────────────────────────────────────────────┐
│          ThreadPoolExecutor 核心参数                      │
│                                                          │
│  public ThreadPoolExecutor(                              │
│      int corePoolSize,         // 核心线程数             │
│      int maximumPoolSize,      // 最大线程数             │
│      long keepAliveTime,       // 非核心线程空闲存活时间  │
│      TimeUnit unit,            // 时间单位               │
│      BlockingQueue<Runnable> workQueue,  // 工作队列     │
│      ThreadFactory threadFactory,          // 线程工厂   │
│      RejectedExecutionHandler handler      // 拒绝策略   │
│  )                                                       │
│                                                          │
│  线程池状态（ctl 变量，高 3 位状态 + 低 29 位线程数）：        │
│  - RUNNING：接受新任务，执行队列任务                      │
│  - SHUTDOWN：不接受新任务，执行队列任务                   │
│  - STOP：不接受新任务，不执行队列任务，中断正在执行的任务   │
│  - TIDYING：所有任务已终止，线程数为 0                      │
│  - TERMINATED：已终止                                     │
└─────────────────────────────────────────────────────────┘
```

### 3.3 核心参数详解表

| 参数 | 作用 | 设置建议 |
|------|------|----------|
| **corePoolSize** | 核心线程数，线程池维持的最小线程数 | CPU 密集型：N+1；IO 密集型：2N |
| **maximumPoolSize** | 最大线程数，线程池允许的最大线程数 | 根据系统资源和队列容量设置 |
| **keepAliveTime** | 非核心线程空闲时的存活时间 | 默认 60 秒，可调整 |
| **workQueue** | 任务队列，存储等待执行的任务 | 有界队列/无界队列，根据场景选择 |
| **threadFactory** | 线程工厂，用于创建新线程 | 自定义线程名、优先级、守护线程 |
| **handler** | 拒绝策略，队列满且线程数达最大时的处理 | 4 种内置策略，可自定义 |

### 3.4 源码分析（execute 方法）

```java
// ThreadPoolExecutor 核心方法
public void execute(Runnable command) {
    if (command == null)
        throw new NullPointerException();

    int c = ctl.get();

    // 步骤 1：核心线程数未满，创建新线程
    if (workerCountOf(c) < corePoolSize) {
        if (addWorker(command, true))  // true=核心线程
            return;
        c = ctl.get();
    }

    // 步骤 2：尝试将任务放入队列
    // 线程池 RUNNING 状态且队列添加成功
    if (isRunning(c) && workQueue.offer(command)) {
        int recheck = ctl.get();
        // 双重检查：如果线程池不是 RUNNING 状态，移除任务并拒绝
        if (!isRunning(recheck) && remove(command))
            reject(command);
        // 如果线程数为 0，创建空转的核心线程
        else if (workerCountOf(recheck) == 0)
            ensurePrestart();
    }
    // 步骤 3：队列已满，创建非核心线程
    else if (!addWorker(command, false)) {  // false=非核心线程
        // 步骤 4：线程数已达最大，执行拒绝策略
        reject(command);
    }
}
```

### 3.5 addWorker 方法（创建线程）

```java
// 添加工作线程
private boolean addWorker(Runnable firstTask, boolean core) {
    retry:
    for (;;) {
        int c = ctl.get();
        int rs = runStateOf(c);

        // 检查线程池状态
        if (rs >= SHUTDOWN &&
            !(rs == SHUTDOWN &&
              firstTask == null &&
              !workQueue.isEmpty()))
            return false;

        for (;;) {
            int wc = workerCountOf(c);
            // 检查线程数是否超过限制
            if (wc >= CAPACITY ||
                wc >= (core ? corePoolSize : maximumPoolSize))
                return false;

            // CAS 增加线程数
            if (compareAndIncrementWorkerCount(c))
                break retry;
            c = ctl.get();
            if (runStateOf(c) != rs)
                continue retry;
        }
    }

    boolean workerStarted = false;
    boolean workerAdded = false;
    Worker w = null;
    try {
        // 创建 Worker 对象
        w = new Worker(firstTask);
        final Thread t = w.thread;
        if (t != null) {
            final ReentrantLock mainLock = this.mainLock;
            mainLock.lock();
            try {
                // 添加到 workers 集合
                workers.add(w);
                int largestPoolSize = largestPoolSize;
                if (s > largestPoolSize)
                    largestPoolSize = s;
                workerAdded = true;
            } finally {
                mainLock.unlock();
            }
            if (workerAdded) {
                // 启动线程
                t.start();
                workerStarted = true;
            }
        }
    } finally {
        if (!workerStarted)
            addWorkerFailed(w);
    }
    return workerStarted;
}
```

### 3.6 Worker 线程执行任务

```java
// Worker 是线程池内部的工作单元
private final class Worker extends AbstractQueuedSynchronizer implements Runnable {
    final Thread thread;
    Runnable firstTask;

    Worker(Runnable firstTask) {
        this.firstTask = firstTask;
        this.thread = getThreadFactory().newThread(this);
    }

    // 线程启动后执行 runWorker 方法
    public void run() {
        runWorker(this);
    }
}

// 工作线程执行逻辑
final void runWorker(Worker w) {
    Thread wt = Thread.currentThread();
    Runnable task = w.firstTask;
    w.firstTask = null;
    w.unlock();  // 允许中断

    boolean completedAbruptly = true;
    try {
        // 执行任务循环
        while (task != null || (task = getTask()) != null) {
            w.lock();  // 防止线程池停止时中断

            // 执行前钩子
            beforeExecute(wt, task);
            Throwable thrown = null;
            try {
                task.run();  // 执行任务
            } catch (Throwable x) {
                thrown = x;
                afterExecute(task, thrown);
            } finally {
                task = null;
                w.completedTasks++;
                w.unlock();
            }
        }
        completedAbruptly = false;
    } finally {
        // 执行后处理
        processWorkerExit(w, completedAbruptly);
    }
}

// 从队列获取任务
private Runnable getTask() {
    boolean timedOut = false;
    for (;;) {
        int c = ctl.get();
        int rs = runStateOf(c);

        // 线程池已停止，返回 null
        if (rs >= SHUTDOWN && (rs >= STOP || workQueue.isEmpty())) {
            decrementWorkerCount(c);
            return null;
        }

        int wc = workerCountOf(c);
        // 是否允许超时（非核心线程或 allowCoreThreadTimeOut）
        boolean timed = allowCoreThreadTimeOut || wc > corePoolSize;

        // 获取任务（阻塞/超时）
        if (wc > corePoolSize || (timed && timedOut)) {
            // 尝试减少线程数
            if (compareAndDecrementWorkerCount(c))
                return null;
            continue;
        }

        try {
            // 超时获取（非核心线程）或阻塞获取（核心线程）
            Runnable r = timed ?
                workQueue.poll(keepAliveTime, TimeUnit.NANOSECONDS) :
                workQueue.take();
            if (r != null)
                return r;
            timedOut = true;
        } catch (InterruptedException retry) {
            timedOut = false;
        }
    }
}
```

## 四、代码示例

### 4.1 线程池基本使用

```java
import java.util.concurrent.*;

public class ThreadPoolExample {
    public static void main(String[] args) {
        // 1. 手动创建线程池（推荐方式）
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            5,                      // 核心线程数
            10,                     // 最大线程数
            60L,                    // 非核心线程空闲存活时间
            TimeUnit.SECONDS,       // 时间单位
            new ArrayBlockingQueue<>(100),  // 工作队列
            new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
        );

        // 2. 提交任务
        for (int i = 0; i < 20; i++) {
            final int taskId = i;
            executor.submit(() -> {
                System.out.println(Thread.currentThread().getName() +
                    " 执行任务：" + taskId);
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            });
        }

        // 3. 关闭线程池
        executor.shutdown();
        try {
            if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
                executor.shutdownNow();
            }
        } catch (InterruptedException e) {
            executor.shutdownNow();
        }
    }
}
```

### 4.2 自定义线程工厂

```java
import java.util.concurrent.*;
import java.util.concurrent.atomic.AtomicInteger;

public class CustomThreadFactory implements ThreadFactory {
    private final AtomicInteger threadNumber = new AtomicInteger(1);
    private final String namePrefix;
    private final boolean daemon;

    public CustomThreadFactory(String namePrefix, boolean daemon) {
        this.namePrefix = namePrefix;
        this.daemon = daemon;
    }

    @Override
    public Thread newThread(Runnable r) {
        Thread t = new Thread(r, namePrefix + "-Thread-" + threadNumber.getAndIncrement());
        t.setDaemon(daemon);
        if (t.getPriority() != Thread.NORM_PRIORITY) {
            t.setPriority(Thread.NORM_PRIORITY);
        }
        return t;
    }

    public static void main(String[] args) {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            5,
            10,
            60L,
            TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(100),
            new CustomThreadFactory("MyPool", false),  // 自定义线程工厂
            new ThreadPoolExecutor.CallerRunsPolicy()
        );

        executor.submit(() -> {
            System.out.println("执行线程：" + Thread.currentThread().getName());
        });
    }
}
```

            final int taskId = i;
            executor.submit(() -> {
                System.out.println("执行任务：" + taskId);
                try {
                    Thread.sleep(2000);
                } catch (InterruptedException e) {}
            });
        }
    }
}

import java.util.concurrent.*;

public class ThreadPoolMonitor {
    public static void main(String[] args) throws InterruptedException {

            System.out.println("最大线程数：" + executor.getMaximumPoolSize());
            System.out.println("当前线程数：" + executor.getPoolSize());

    }

## 五、常见面试题扩展

### 扩展 1：线程池的 7 个参数分别是什么意思？

**答：**

| 参数 | 含义 | 备注 |
|------|------|------|
| corePoolSize | 核心线程数 | 线程池维持的最小线程数，即使空闲也不会被回收（除非 allowCoreThreadTimeOut） |
| maximumPoolSize | 最大线程数 | 线程池允许创建的最大线程数 |
| keepAliveTime | 空闲存活时间 | 非核心线程空闲超过此时间会被回收 |
| unit | 时间单位 | keepAliveTime 的单位 |
| workQueue | 工作队列 | 存储等待执行的任务，常用 ArrayBlockingQueue、LinkedBlockingQueue |
| threadFactory | 线程工厂 | 创建线程，可自定义线程名、优先级等 |
| handler | 拒绝策略 | 队列满且线程数达最大时的处理策略 |

### 扩展 2：线程池有哪些拒绝策略？

**答：**

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| **AbortPolicy** | 抛 RejectedExecutionException 异常 | 默认策略，需要感知任务被拒绝 |
| **CallerRunsPolicy** | 调用者线程执行任务 | 降低提交速度，避免任务丢失 |
| **DiscardPolicy** | 直接丢弃任务，不抛异常 | 允许任务丢失的场景 |
| **DiscardOldestPolicy** | 丢弃队列中最老的任务，然后重试 | 新任务比旧任务重要 |

**自定义拒绝策略：**

        // 记录日志、发送告警、持久化任务等

    }

### 扩展 3：如何合理设置线程池参数？

**答：**

```
1. CPU 密集型任务
   - 特点：计算密集，CPU 利用率高
   - 配置：corePoolSize = CPU 核数 + 1
   - 理由：减少线程切换，+1 防止线程意外挂起

2. IO 密集型任务
   - 特点：大量 IO 操作，CPU 等待
   - 配置：corePoolSize = CPU 核数 * 2
   - 或：corePoolSize = CPU 核数 / (1 - 阻塞系数)
   - 理由：充分利用 CPU 等待时间

3. 混合型任务
   - 根据任务特性拆分：CPU 密集型 + IO 密集型
   - 分别配置不同的线程池

4. 考虑因素
   - 系统内存：线程数过多会导致内存不足
   - 任务队列：有界队列防止 OOM
   - 最大线程数：根据系统资源设置上限
```

### 扩展 4：线程池如何正确关闭？

**答：**

```java
// 1. 优雅关闭（推荐）
executor.shutdown();  // 不再接受新任务
try {
    // 等待所有任务完成
    if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
        executor.shutdownNow();  // 强制关闭
    }
} catch (InterruptedException e) {
    executor.shutdownNow();
    Thread.currentThread().interrupt();
}

// 2. 立即关闭（不推荐，可能丢失任务）
executor.shutdownNow();  // 中断正在执行的任务

// shutdown() vs shutdownNow() 区别
shutdown():
- 不再接受新任务
- 等待队列中的任务执行完成
- 不中断正在执行的任务

shutdownNow():
- 不再接受新任务
- 丢弃队列中的任务（返回未执行的任务列表）
- 尝试中断正在执行的任务（通过 Thread.interrupt()）
```

### 扩展 5：线程池的常见陷阱有哪些？

**答：**

```
1. 使用 Executors 创建线程池（不推荐）
   - newFixedThreadPool：队列无界，可能 OOM
   - newSingleThreadExecutor：队列无界，可能 OOM
   - newCachedThreadPool：线程数无界，可能 OOM
   - 建议：手动创建 ThreadPoolExecutor

2. 队列容量设置过大
   - 队列积压过多任务，内存溢出
   - 建议：使用有界队列，设置合理容量

3. 线程数设置过大
   - 线程切换开销大，性能下降
   - 建议：根据任务类型和系统资源设置

4. 不监控线程池状态
   - 无法及时发现线程池异常
   - 建议：定期监控 activeCount、queueSize 等指标

5. 任务执行时间过长
   - 占用线程，导致其他任务等待
   - 建议：任务超时控制、异步处理

6. 线程池未正确关闭
   - 资源泄漏，应用无法退出
   - 建议：finally 块中关闭，或使用 try-with-resources
```

## 六、面试答题话术

**面试官问：请说说线程池的任务执行流程？**

**标准回答：**

> 线程池任务执行流程分为 4 个步骤：
>
> 1. **核心线程数未满**：创建新核心线程执行任务
> 2. **核心线程已满**：将任务放入工作队列等待
> 3. **队列已满**：创建非核心线程执行任务
> 4. **线程数达最大**：执行拒绝策略
>
> 线程池有 7 个核心参数：corePoolSize、maximumPoolSize、keepAliveTime、unit、workQueue、threadFactory、handler。
>
> 拒绝策略有 4 种：AbortPolicy（抛异常）、CallerRunsPolicy（调用者执行）、DiscardPolicy（直接丢弃）、DiscardOldestPolicy（丢弃最老任务）。

**加分回答：**

> 补充几点：
> 1. 线程池状态有 5 种：RUNNING、SHUTDOWN、STOP、TIDYING、TERMINATED
> 2. 核心线程默认不会被回收，除非设置 allowCoreThreadTimeOut=true
> 3. 非核心线程空闲超过 keepAliveTime 会被回收
> 4. 建议使用 ThreadPoolExecutor 手动创建线程池，避免使用 Executors
> 5. 线程池参数设置：CPU 密集型 N+1，IO 密集型 2N
> 6. 关闭线程池用 shutdown() + awaitTermination()，避免任务丢失
> 7. Worker 线程执行完任务后，会从队列 take() 或 poll() 获取新任务

## 七、速记表格

| 步骤 | 条件 | 动作 |
|------|------|------|
| 1 | 线程数 < corePoolSize | 创建核心线程 |
| 2 | 核心线程已满 | 放入工作队列 |
| 3 | 队列已满 & 线程数 < maximumPoolSize | 创建非核心线程 |
| 4 | 队列已满 & 线程数 = maximumPoolSize | 拒绝策略 |

| 参数 | 默认值/常用值 | 说明 |
|------|--------------|------|
| corePoolSize | 根据任务类型设置 | 核心线程数 |
| maximumPoolSize | 根据资源设置 | 最大线程数 |
| keepAliveTime | 60 秒 | 非核心线程存活时间 |
| workQueue | ArrayBlockingQueue | 有界队列 |
| handler | AbortPolicy | 拒绝策略 |
