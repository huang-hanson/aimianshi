# 场景题：AI 项目的实际应用场景有哪些？

## 一、核心结论

**AI 项目（尤其是 LLM）的实际应用场景可分为 6 大领域：**

| 领域 | 典型场景 | 技术栈 | 商业价值 |
|------|---------|--------|----------|
| **智能客服** | 客服机器人、工单处理、智能问答 | RAG + 意图识别 | 降本增效 |
| **内容生成** | 文案生成、报告生成、代码生成 | LLM + Prompt 工程 | 提高生产力 |
| **知识管理** | 企业知识库、文档检索、智能助手 | RAG + 向量检索 | 知识沉淀 |
| **数据分析** | BI 分析、数据洞察、自动报表 | Text-to-SQL + 可视化 | 数据驱动 |
| **办公提效** | 会议助手、邮件助手、文档总结 | 语音识别 + 文本摘要 | 提升效率 |
| **垂直应用** | 医疗、法律、金融、教育 | 领域微调 + 专业 RAG | 行业赋能 |

## 三、场景详解

### 3.1 智能客服 ⭐⭐⭐

**痛点：**
- 人工客服成本高（7×24 小时值班）
- 响应速度慢（排队等待）
- 服务质量不稳定（人员流动大）

**AI 解决方案：**

```
用户问题
    │
    ▼
┌─────────────────┐
│  意图识别       │  ← 分类：咨询/投诉/售后
└────┬────────────┘
     │
     ▼
┌─────────────────┐
│  知识库检索     │  ← RAG 检索相关文档
└────┬────────────┘
     │
     ▼
┌─────────────────┐
│  LLM 生成答案    │  ← 基于检索结果生成
└────┬────────────┘
     │
     ▼
┌─────────────────┐
│  人工兜底       │  ← 置信度低时转人工
└─────────────────┘
```

**技术架构：**

```python
from langchain.chat_models import ChatOpenAI
from langchain.vectorstores import Milvus
from langchain.retrievers import ContextualCompressionRetriever

class CustomerServiceBot:
    """智能客服机器人"""
    
    def __init__(self):
        # 1. 初始化 LLM
        self.llm = ChatOpenAI(model="gpt-4", temperature=0.3)
        
        # 2. 初始化知识库（产品文档、FAQ、历史工单）
        self.vectorstore = Milvus.from_existing(
            collection_name="customer_service_kb",
            embedding_function=self.embeddings
        )
        
        # 3. 意图分类器
        self.intent_classifier = IntentClassifier()
        
        # 4. 人工兜底阈值
        self.handoff_threshold = 0.6
    
    def handle_query(self, user_query: str, chat_history: list = None):
        # 1. 意图识别
        intent = self.intent_classifier.classify(user_query)
        
        # 2. 根据意图路由
        if intent == "complaint":
            # 投诉类，直接转人工
            return self.handoff_to_human(user_query)
        
        elif intent == "inquiry":
            # 咨询类，RAG 检索
            docs = self.vectorstore.similarity_search(user_query, k=3)
            context = "\n".join([d.page_content for d in docs])
            
            # 3. 生成回答
            prompt = f"""基于以下产品信息回答用户问题：

{context}

用户问题：{user_query}

请用友好、专业的语气回答："""
            
            response = self.llm.invoke(prompt)
            
            # 4. 检查置信度
            if self.get_confidence(response) < self.handoff_threshold:
                return self.handoff_to_human(user_query)
            
            return response
        
        elif intent == "feedback":
            # 反馈类，记录工单
            return self.create_ticket(user_query)
    
    def handoff_to_human(self, query: str):
        """转人工客服"""
        # 创建工单，通知人工客服
        ticket_id = create_ticket(query, priority="high")
        return f"您的问题已记录（工单号：{ticket_id}），人工客服将在 10 分钟内联系您。"
```

**典型客户：**
- 电商平台（淘宝、京东）
- 银行/保险（招行、平安）
- 电信运营商（移动、联通）
- SaaS 企业（钉钉、企业微信）

**商业价值：**
- 客服成本降低 60-80%
- 响应时间从分钟级降到秒级
- 24 小时不间断服务

- 文案/报告撰写耗时

**AI 解决方案：**

| 内容类型 | 输入 | 输出 | 应用场景 |
|---------|------|------|---------|
| **营销文案** | 产品信息 + 目标受众 | 广告语、推文 | 电商、品牌营销 |
| **技术文档** | 代码/API 定义 | 使用文档、README | 软件开发 |
| **业务报告** | 数据 + 分析维度 | 分析报告、PPT | 运营、财务 |
| **代码生成** | 需求描述 | 可运行代码 | 软件开发 |
| **多语言翻译** | 源语言文本 | 目标语言文本 | 跨境电商、出海 |

### 3.4 数据分析 ⭐⭐

**痛点：**
- 数据分析门槛高（需要会 SQL）
- 报表开发周期长
- 业务人员无法自助分析

**AI 解决方案：Text-to-SQL**

```python
from langchain.sql_database import SQLDatabase
from langchain.agents import create_sql_agent

class DataAnalysisBot:
    """数据分析助手"""
    
    def __init__(self, db_uri: str):
        # 连接数据库
        self.db = SQLDatabase.from_uri(db_uri)
        
        # 创建 SQL Agent
        self.agent = create_sql_agent(
            llm=ChatOpenAI(model="gpt-4"),
            database=self.db,
            verbose=True
        )
    
    def query(self, natural_language_query: str):
        """用自然语言查询数据"""
        
        prompt = f"""你是一个数据分析助手。
用户问题：{natural_language_query}

可用的数据库表：
- users (user_id, name, register_date, city)
- orders (order_id, user_id, amount, order_date, status)
- products (product_id, name, category, price)

请用 SQL 查询并回答问题："""
        
        return self.agent.run(prompt)

# 使用示例
bot = DataAnalysisBot("mysql+pymysql://user:pass@localhost/sales")

# 用户问：上个月销售额 Top 5 的产品是哪些？
```

**痛点：**

**AI 解决方案：**

| 场景 | 功能 | 技术 |
|------|------|------|
| **会议助手** | 语音转文字、会议纪要、待办提取 | ASR + 文本摘要 |
| **邮件助手** | 邮件分类、自动回复、优先级排序 | 意图识别 + 文本生成 |
| **文档总结** | 长文档摘要、关键信息提取 | 文本摘要 + NER |
| **日程管理** | 智能排程、会议冲突检测 | 规则引擎 + 优化算法 |

**会议纪要示例：**

```python
class MeetingAssistant:
    """会议助手"""
    
    def __init__(self):
        # 语音识别
        self.asr = WhisperASR()
        # LLM
        self.llm = ChatOpenAI(model="gpt-4")
    
    def process_meeting(self, audio_file: str):
        # 1. 语音转文字
        transcript = self.asr.transcribe(audio_file)
        
        # 2. 发言者分离
        speakers = self.diarize_speakers(transcript)
        
        # 3. 生成会议纪要
        summary_prompt = f"""请为以下会议记录生成纪要：

{speakers}

输出格式：
## 会议主题
## 参会人员
## 主要讨论内容
## 决议事项
## 待办事项（负责人 + 截止日期）
"""
        summary = self.llm.invoke(summary_prompt)
        
        # 4. 提取待办事项
        action_items = self.extract_action_items(speakers)
        
        return {
            "summary": summary,
            "action_items": action_items,
            "recording": audio_file,
            "transcript": transcript
        }
```

**典型客户：**

**商业价值：**

## 五、面试答题话术

**面试官问：AI 项目的实际应用场景有哪些？**

**答：** 我主要从 6 个方向来看：

**第一，智能客服**，这是最成熟的场景。用 RAG 检索知识库 + LLM 生成回答，置信度低时转人工。客户包括电商、银行、电信。商业价值是客服成本降低 60-80%。

**第二，内容生成**，包括营销文案、技术文档、代码生成、多语言翻译。用 LLM + Prompt 工程实现。客户有媒体、广告公司、出海企业。内容产出效率提升 3-5 倍。

**第三，知识管理**，解决企业知识分散、检索难、沉淀难的问题。用 RAG + 向量检索实现语义搜索。大型企业和律所是典型客户。

**第四，数据分析**，Text-to-SQL让业务人员自助分析。用 SQL Agent 实现自然语言查数。互联网公司和金融机构需求强烈。

**第五，办公提效**，会议助手、邮件助手、文档总结。用 ASR+ 文本摘要技术。普惠型应用，易推广。

**第六，垂直应用**，医疗、法律、金融、教育。需要领域知识 + 专业 RAG。价值高但门槛也高。

**落地建议：** 从智能客服快速切入，积累经验和案例，再横向扩展到知识管理和内容生成。技术选型上，POC 用 OpenAI API，国内生产用通义/文心，有隐私需求时用开源模型私有部署。

## 六、扩展问题

### 扩展 1：如何评估 AI 项目的 ROI？

**答：** 从三个维度：
1. **成本节约** - 人力成本降低（客服、内容创作）
2. **效率提升** - 时间节省（数据分析、办公提效）
3. **收入增长** - 转化率提升（个性化推荐、营销文案）

### 扩展 2：AI 项目落地最大的挑战是什么？

**答：** 三个挑战：
1. **数据安全** - 企业担心数据泄露 → 私有部署
2. **幻觉问题** - LLM 胡说八道 → RAG+ 人工审核
3. **集成难度** - 与企业现有系统集成 → API+ 中间件

### 扩展 3：中小企业如何选择 AI 服务商？

**答：** 四个考量：
1. **模型能力** - 效果是否满足需求
2. **数据安全** - 是否支持私有部署
3. **集成能力** - 是否容易对接现有系统
4. **成本** - API 调用费 vs 私有部署费
