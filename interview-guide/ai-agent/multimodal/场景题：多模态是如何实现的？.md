# 场景题：多模态是如何实现的？

## 一、核心结论

**多模态 AI 的核心实现方式分为三类：**

| 类型 | 输入 | 输出 | 典型应用 |
|------|------|------|----------|
| **图生文** | 图片 | 文本 | 图像描述、OCR、视觉问答 |
| **文生图** | 文本 | 图片 | DALL-E、Midjourney、Stable Diffusion |
| **多模态理解** | 图片 + 文本 | 文本 | GPT-4V、Claude、Gemini |

**核心技术：**
1. **CLIP** - 图文对齐，连接视觉和语言
2. **Transformer** - 统一处理多模态数据
3. **Diffusion** - 图像生成的主流方法
4. **ViT** - 将图片转换为 Token 序列

## 三、图生文（Image-to-Text）实现

### 3.1 技术原理

```
图片 → ViT 编码器 → 图像 Embedding → 投影层 → LLM → 文本输出
```

**核心步骤：**
1. **ViT 编码**：将图片切分为 16×16 的 Patch，每个 Patch 转换为一个向量
2. **模态对齐**：通过线性投影将图像向量映射到 LLM 的词向量空间
3. **LLM 生成**：LLM 将图像向量当作特殊的"Token"，自回归生成文本

### 3.2 典型架构对比

| 模型 | 视觉编码器 | 对齐方式 | LLM |
|------|-----------|---------|-----|
| **Flamingo** | ViT | Perceiver Resampler | Chinchilla |
| **BLIP-2** | ViT | Q-Former | OPT/Vicuna |
| **LLaVA** | ViT | 线性投影 | Vicuna/Llama |
| **GPT-4V** | 未知 | 未知 | GPT-4 |

### 3.3 LLaVA 实现代码

```python
import torch
import torch.nn as nn
from transformers import CLIPVisionModel, LlamaForCausalLM, LlamaTokenizer

class LLaVAModel(nn.Module):
    """LLaVA 多模态模型实现"""
    
    def __init__(self, vision_model_name, llm_model_name):
        super().__init__()
        
        # 1. 视觉编码器（CLIP ViT）
        self.vision_tower = CLIPVisionModel.from_pretrained(vision_model_name)
        
        # 2. 大语言模型
        self.llm = LlamaForCausalLM.from_pretrained(llm_model_name)
        self.tokenizer = LlamaTokenizer.from_pretrained(llm_model_name)
        
        # 3. 投影层：将 ViT 输出映射到 LLM 的词向量空间
        # CLIP ViT 输出维度：1024
        # Llama 词向量维度：4096
        self.projector = nn.Linear(1024, 4096)
    
    def encode_image(self, images):
        """将图片编码为向量"""
        # images: [batch_size, 3, 224, 224]
        with torch.no_grad():
            # ViT 编码
            image_features = self.vision_tower(images, output_hidden_states=True)
            # 取最后一层的输出
            image_features = image_features.hidden_states[-1]  # [B, 257, 1024]
        
        # 投影到 LLM 维度
        image_embeddings = self.projector(image_features)  # [B, 257, 4096]
        return image_embeddings
    
    def forward(self, images, input_ids, attention_mask=None, labels=None):
        """
        前向传播
        
        Args:
            images: 图片输入 [B, 3, 224, 224]
            input_ids: 文本输入，包含特殊占位符 <image>
            attention_mask: 注意力掩码
            labels: 训练时的标签
        """
        # 1. 编码图片
        image_embeddings = self.encode_image(images)
        
        # 2. 获取文本的嵌入
        text_embeddings = self.llm.get_input_embeddings()(input_ids)
        
        # 3. 将图片嵌入插入到文本嵌入中 <image> 的位置
        # 假设 <image> 的 token id 是 128256
        image_token_id = 128256
        image_positions = (input_ids == image_token_id).nonzero()
        
        # 构造新的输入嵌入
        inputs_embeds = text_embeddings
        for batch_idx, seq_idx in image_positions:
            # 将 <image> 位置的嵌入替换为图片嵌入
            inputs_embeds[batch_idx, seq_idx:seq_idx+257] = image_embeddings[batch_idx]
        
        # 4. 通过 LLM 生成
        outputs = self.llm(
            inputs_embeds=inputs_embeds,
            attention_mask=attention_mask,
            labels=labels
        )
        
        return outputs
```

### 3.4 使用示例

```python
from PIL import Image
import requests
from transformers import CLIPProcessor

# 初始化模型
model = LLaVAModel("openai/clip-vit-large-patch14", "vicuna-7b")
model.eval()

# 加载图片
image = Image.open("cat.jpg")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-large-patch14")

# 处理图片
inputs = processor(images=image, return_tensors="pt")
images = inputs["pixel_values"]

# 构造输入文本

input_ids = model.tokenizer.encode(prompt, return_tensors="pt")

# 生成回答

    outputs = model.llm.generate(
        input_ids=input_ids,
        max_new_tokens=100
    )

print(response)
# 输出："这是一只橘色的猫，正躺在沙发上..."
```

## 四、文生图（Text-to-Image）实现

### 4.1 Diffusion 模型原理

**核心思想：** 模拟物理扩散过程
- **前向过程**：逐步加噪声，直到图片变成纯噪声
- **反向过程**：逐步去噪声，从噪声中生成图片

```
文本 → 文本编码器 → 文本 Embedding → U-Net → 去噪 → 解码器 → 图片
                              ↓
                         条件引导
```

### 4.2 Stable Diffusion 架构

```python
import torch
from diffusers import StableDiffusionPipeline, DPMSolverMultistepScheduler

# 1. 加载预训练模型
model_id = "runwayml/stable-diffusion-v1-5"

# 2. 使用 DPM Solver 加速采样（从 50 步降到 20 步）
pipe = StableDiffusionPipeline.from_pretrained(model_id)
pipe.scheduler = DPMSolverMultistepScheduler.from_config(pipe.scheduler.config)
pipe = pipe.to("cuda")

# 3. 生成图片
prompt = "一只可爱的橘猫，在阳光下，高清，4K"
negative_prompt = "模糊，低质量，变形"

image = pipe(
    prompt=prompt,
    negative_prompt=negative_prompt,
    num_inference_steps=20,  # 采样步数
    guidance_scale=7.5,       # 引导系数，越大越贴近 prompt
    height=512,
    width=512
).images[0]

image.save("generated_cat.png")
```

### 4.3 Diffusion 核心代码解析

```python
import torch
import torch.nn as nn

class UNet2DConditionModel(nn.Module):
    """Stable Diffusion 的 U-Net 架构"""
    
    def __init__(self):
        super().__init__()
        
        # 下采样
        self.down_blocks = nn.ModuleList([
            DownBlock2D(...),
            CrossAttnDownBlock2D(...)  # 带 Cross-Attention
        ])
        
        # 中间层
        self.mid_block = UNetMidBlock2DCrossAttn(...)
        
        # 上采样
        self.up_blocks = nn.ModuleList([
            CrossAttnUpBlock2D(...),
            UpBlock2D(...)
        ])
    
    def forward(self, x, t, encoder_hidden_states):
        """
        Args:
            x: 当前时刻的噪声图片 [B, 4, H, W]
            t: 时间步 embedding
            encoder_hidden_states: 文本编码 [B, seq_len, dim]
        """
        # 下采样
        for down_block in self.down_blocks:
            x = down_block(x, encoder_hidden_states=encoder_hidden_states)
        
        # 中间层
        x = self.mid_block(x, encoder_hidden_states=encoder_hidden_states)
        
        # 上采样
        for up_block in self.up_blocks:
            x = up_block(x, encoder_hidden_states=encoder_hidden_states)
        
        return x


class CrossAttention(nn.Module):
    """Cross-Attention 层：文本条件引导"""
    
    def __init__(self, query_dim, context_dim, heads=8):
        super().__init__()
        self.to_q = nn.Linear(query_dim, query_dim)
        self.to_k = nn.Linear(context_dim, query_dim)
        self.to_v = nn.Linear(context_dim, query_dim)
        self.to_out = nn.Linear(query_dim, query_dim)
    
    def forward(self, x, context):
        """
        Args:
            x: 图片特征（Query）
            context: 文本特征（Key, Value）
        """
        q = self.to_q(x)
        k = self.to_k(context)
        v = self.to_v(context)
        
        # 缩放点积注意力
        scale = q.shape[-1] ** -0.5
        attn = torch.softmax(q @ k.transpose(-2, -1) * scale, dim=-1)
        out = attn @ v
        
        return self.to_out(out)
```

### 4.4 采样过程详解

```python
@torch.no_grad()
def diffusion_sampling(model, text_embeddings, steps=50):
    """Diffusion 采样过程"""
    
    # 1. 从标准正态分布采样初始噪声
    latents = torch.randn(1, 4, 64, 64).cuda()
    
    # 2. 设置采样调度器
    scheduler = DPMSolverMultistepScheduler(
        num_train_timesteps=1000,
        beta_start=0.00085,
        beta_end=0.012
    )
    
    # 3. 逐步去噪
    scheduler.set_timesteps(steps)
    
    for t in scheduler.timesteps:
        # 预测噪声
        noise_pred = model(latents, t, encoder_hidden_states=text_embeddings)
        
        # 无 classifier 引导
        # 一半用空 prompt，一半用真实 prompt
        noise_cond = noise_pred[1:]  # 有条件
        noise_uncond = noise_pred[:1]  # 无条件
        # CFG 公式
        guidance_scale = 7.5
        noise_pred = noise_uncond + guidance_scale * (noise_cond - noise_uncond)
        
        # 更新 latents
        latents = scheduler.step(noise_pred, t, latents).prev_sample
    
    # 4. 解码到像素空间
    # latents: [1, 4, 64, 64] → image: [1, 3, 512, 512]
    image = decode_latents(latents)
    
    return image
```

## 五、多模态对齐技术：CLIP

### 5.1 CLIP 原理

**CLIP（Contrastive Language-Image Pre-training）** 是 OpenAI 提出的图文对比学习框架。

```
图片 ──→ ViT ──→ 图片 Embedding ──→ 归一化 ──┐
                                             │ → 对比损失
文本 ──→ Transformer ──→ 文本 Embedding ──→ 归一化 ──┘
```

**核心思想：**
- 将图片和对应的文本描述编码到同一个向量空间
- 匹配的图文对相似度更高，不匹配的相似度更低

### 5.2 CLIP 代码实现

```python
import torch
import torch.nn as nn
from transformers import CLIPModel, CLIPProcessor

# 加载预训练 CLIP
model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

from PIL import Image

# 准备输入
image = Image.open("cat.jpg")
texts = ["一只猫", "一只狗", "一辆汽车"]

# 处理输入
inputs = processor(
    text=texts,
    images=image,
    return_tensors="pt",
    padding=True
)

# 获取特征
outputs = model(**inputs)
image_embeds = outputs.image_embeds      # [1, 512]
text_embeds = outputs.text_embeds        # [3, 512]

# 计算相似度
logits_per_image = outputs.logits_per_image  # [1, 3]
probs = logits_per_image.softmax(dim=1)      # 概率分布

print(f"是猫的概率：{probs[0][0]:.2%}")
print(f"是狗的概率：{probs[0][1]:.2%}")
print(f"是汽车的概率：{probs[0][2]:.2%}")
```

### 5.3 CLIP 的应用场景

| 应用 | 说明 |
|------|------|
| **零样本分类** | 无需训练，直接用文本描述作为类别 |
| **图文检索** | 用文本搜图片，或用图片搜文本 |
| **多模态理解** | 作为 ViT 编码器，用于 LLaVA 等模型 |

## 七、面试答题话术

**面试官问：多模态是如何实现的？**

**答：** 我从三个方向来回答：

**第一，图生文（Image-to-Text）：** 核心是用 ViT 将图片编码为向量，通过投影层映射到 LLM 的词向量空间，LLM 将这些向量当作特殊 Token 自回归生成文本。代表作品有 LLaVA、BLIP-2、GPT-4V。

**第二，文生图（Text-to-Image）：** 主流是 Diffusion 模型，通过逐步去噪声从随机噪声生成图片，文本通过 Cross-Attention 层引导生成方向。代表作品有 Stable Diffusion、DALL-E 3、Midjourney。

**第三，多模态对齐：** 用 CLIP 对比学习将图文编码到同一向量空间，实现零样本分类和图文检索。

**技术栈：**
- **视觉编码器**：ViT、CLIP
- **对齐方式**：线性投影、Q-Former、Perceiver Resampler
- **LLM**：Llama、Vicuna
- **图像生成**：Diffusion + U-Net + Cross-Attention

**举个例子**：LLaVA 的实现是用 CLIP ViT 编码图片得到 257 个 1024 维向量，通过线性投影映射到 4096 维，插入到 Llama 的输入序列中，替换掉 `<image>` 占位符，然后自回归生成回答。

## 八、扩展问题

### 扩展 1：如何处理多张图片？

**答：** 三种方案：
1. **拼接**：将多张图片的向量序列拼接成长序列
2. **池化**：对多张图片的向量做 Attention Pooling
3. **Q-Former**：用 BLIP-2 的 Q-Former 压缩多张图片信息

### 扩展 2：如何提高多模态模型的回答质量？

**答：** 四个优化点：
1. **高质量图文对训练**：用更高质量的图文对微调
2. **指令微调**：用多模态指令数据 SFT
3. **RLHF**：用人类反馈强化学习对齐
4. **RAG**：结合外部知识库增强

### 扩展 3：Diffusion 模型为什么比 GAN 好？

**答：** 三个优势：
1. **训练稳定**：不需要对抗训练，不会模式坍塌
2. **生成质量高**：细节更丰富，多样性更好
3. **可控性强**：容易做条件引导和编辑
